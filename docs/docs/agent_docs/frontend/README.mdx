## Quick Start

### Development Environment Setup

1. **Open in Dev Container** (Recommended):
   ```bash
   # VS Code will prompt to reopen in container
   # All services start automatically
   # Frontend: http://localhost:9000
   # Backend: http://localhost:8088
   ```

2. **Manual Setup**:
   ```bash
   cd superset-frontend
   npm install
   npm run dev-server
   ```

### Common Development Tasks

**Running Tests:**
```bash
npm run test          # Run all tests
npm run tdd           # Test-driven development mode
npm run cover         # Generate coverage report
```

**Code Quality:**
```bash
npm run lint          # ESLint + TypeScript checking
npm run lint-fix      # Auto-fix linting issues
npm run format        # Format code with Prettier
npm run type          # TypeScript type checking
```

**Building:**
```bash
npm run build         # Production build
npm run build-dev     # Development build
npm run bundle-stats  # Bundle size analysis
```

**Component Development:**
```bash
npm run storybook     # Start Storybook for component development
```

## Architecture Summary

The Superset frontend is built as a **modern React application** with:

- **Plugin-Driven Architecture**: Extensible visualization system
- **Design System**: Consistent UI components based on Ant Design
- **TypeScript**: Strict typing throughout the application
- **Redux State Management**: Centralized state with domain-specific slices
- **Modern Build System**: Webpack 5 with optimization and hot reload
- **Comprehensive Testing**: Unit, integration, and E2E test coverage

## Development Standards

### Component Development
- Use TypeScript for all new components
- Follow Atomic Design principles
- Include Storybook stories for documentation
- Write comprehensive unit tests
- Follow accessibility best practices

### Plugin Development
- Extend the ChartPlugin base class
- Implement proper TypeScript interfaces
- Include control panel configuration
- Add comprehensive tests
- Follow performance best practices

### Code Quality
- ESLint configuration with custom rules
- Prettier formatting with consistent style
- Pre-commit hooks for automated quality checks
- TypeScript strict mode enabled
- Test coverage monitoring
